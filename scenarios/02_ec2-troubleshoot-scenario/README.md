# EC2 Troubleshoot Scenario

## Overview
This lab provides hands-on experience in **troubleshooting Amazon EC2 instances**. It covers common operational issues including connectivity failures, IAM permission errors, and instance health check problems. Perfect for Cloud Support and CloudOps practice.

## Objectives
By the end of this scenario, you will be able to:
- Identify and resolve EC2 connectivity issues (SSH, RDP, etc.)
- Troubleshoot IAM roles and permission errors affecting EC2
- Analyze system and application logs for root cause identification
- Validate EC2 instance health and configuration

## Prerequisites
- AWS account with sufficient permissions for EC2, VPC, IAM, and CloudWatch
- Basic knowledge of EC2, IAM roles, and networking in AWS
- AWS CLI installed and configured
